# Remove Self-Service Sign-Up

**Date:** 2026-03-31
**Status:** Approved

## Goal

Remove all self-service sign-up functionality from the app. BootRoom is invite-only; new members join via an invite link generated by a league admin. There is no reason for a public-facing registration flow.

This is a clean-slate exercise — the new invite-based sign-up flow will be designed separately. This spec covers removal only.

## Scope

### What is removed

| File | Change |
|---|---|
| `components/AuthDialog.tsx` | Remove sign-up tab, username field, password-confirm field, and sign-up submission logic. Dialog retains two modes: sign-in and forgot-password. |
| `app/api/auth/sign-up/route.ts` | Delete entirely. |
| `components/ui/navbar.tsx` | Remove any "Join" / "Sign up" CTA. Keep "Log in" button. |
| Any call site opening `AuthDialog` in `sign-up` mode | Change to open in `sign-in` mode instead. |

### What is not touched

- Sign-in, sign-out, password reset, update-password API routes
- `claim_profile()` RPC (still needed on sign-in)
- Invite link generation UI in league settings
- `accept_game_invite` RPC and invite token infrastructure
- Middleware auth logic

## Approach

Surgical removal only. No refactoring of the AuthDialog structure, no new pages, no architectural changes. The goal is a minimal diff that leaves the codebase ready for a fresh invite-based sign-up design.

## Out of scope

- Designing or building the new invite-based sign-up flow
- Refactoring AuthDialog into a dedicated `/sign-in` page
- Changes to the invite link generation or token infrastructure
