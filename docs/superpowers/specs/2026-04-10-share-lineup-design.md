# Share Lineup — Design Spec

**Date:** 2026-04-10
**Branch:** awmloveland/share-lineup-message

---

## Overview

After an admin saves a lineup to a scheduled week, any league member can share the teams with the group via a single tap. The share delivers the team lists, EWTPI scores, and win prediction as formatted text in any messaging app — WhatsApp, iMessage, Slack, etc. — with a link back to the league page.

---

## Scope

- Formatted text share (no image generation)
- Share button lives in the scheduled week card (`NextMatchCard.tsx`) footer — not in the Lineup Lab tab
- Visible to all members once a lineup has been saved (both `team_a` and `team_b` are non-empty)
- Individual player ratings are never exposed in the share text

---

## Share Text Format

```
⚽ {leagueName} — Week {n}
📅 {day} {date} · {format}

🔵 Team A ({team_a_rating})
{player1}, {player2}, ...

🟣 Team B ({team_b_rating})
{player1}, {player2}, ...

📊 {prediction}

🔗 craft-football.com/{leagueId}
```

**Field sources:**
- `leagueName` — from the `games` table (`name` column), passed as a **new prop** to `NextMatchCard` (currently only `gameId` is passed; callers must also supply `leagueName`)
- `week` — `weeks.week`
- `day` / `date` — derived from `weeks.date` (formatted as e.g. "Thu 10 Apr")
- `format` — `weeks.format` (e.g. "6-a-side")
- `team_a` / `team_b` — `weeks.team_a[]`, `weeks.team_b[]` joined as comma-separated strings
- `team_a_rating` / `team_b_rating` — `weeks.team_a_rating`, `weeks.team_b_rating`, rounded to 1 decimal place
- `prediction` — generated by calling `winProbability(team_a_rating, team_b_rating)` then `winCopy(prob).text`, both already in `lib/utils.ts`
- `leagueId` — same as the existing `gameId` prop in `NextMatchCard`, used to construct the public URL

## New Utility: `buildShareText()`

Added to `lib/utils.ts`. Pure function, no side effects.

```ts
export function buildShareText(params: {
  leagueName: string
  leagueId: string
  week: number
  date: string        // ISO date string, e.g. "2026-04-10"
  format: string
  teamA: string[]
  teamB: string[]
  teamARating: number
  teamBRating: number
}): string
```

Internally calls `winProbability()` and `winCopy()` to produce the prediction line.

## Share Mechanism

Detection logic (in the share button click handler):

```ts
if (navigator.share && window.innerWidth < 768) {
  // Mobile — native share sheet
  await navigator.share({ text })
} else {
  // Desktop — copy to clipboard
  await navigator.clipboard.writeText(text)
  // show "Lineup copied!" toast
}
```

- The native share sheet handles its own feedback on mobile
- On desktop, show a brief toast: **"Lineup copied!"** (2 seconds, same pattern as existing clipboard toast in `LeagueJoinArea.tsx`)
- If `navigator.clipboard` is unavailable (rare), fall back silently — no error shown to user

---

## UI — Button Placement

- **Component:** `NextMatchCard.tsx`, in the existing lineup footer section (already rendered when lineup is saved)
- **Element:** Icon button using `Share2` from `lucide-react`
- **Label:** "Share" (visible text on desktop, icon-only on mobile per existing button patterns)
- **Condition:** Only rendered when `teamA.length > 0 && teamB.length > 0`
- **Role gate:** None — all members see it

---

## Link Destination

`https://craft-football.com/{leagueId}`

Links to the public league page. Lineup Lab (`/{leagueId}/lineup-lab`) requires auth and is not linked.

---

## What Is Not In Scope

- Image generation
- Sharing from the Lineup Lab tab
- Per-player rating display in share text
- Sharing played/cancelled weeks (scheduled weeks only)
- Shareable permalink for a specific lineup
