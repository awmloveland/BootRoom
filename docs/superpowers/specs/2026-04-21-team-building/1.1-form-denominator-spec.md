# 1.1 Form Denominator Fix — Spec
_Date: 2026-04-21_
_Umbrella: [../2026-04-21-team-building-review-design.md](../2026-04-21-team-building-review-design.md)_
_Status: Ready to implement_

## Problem

`wprScore` in `lib/utils.ts` (around lines 96–104) computes the form denominator `maxFormScore` by summing `3 × (1 − i × 0.15)` across all 5 form slots — including `'-'` slots that represent games the player did not exist for or did not attend. This treats dashes as if they were potential wins the player missed, which unfairly penalises anyone with a short playing history.

A player with `recentForm = 'WW---'` (2 games, both won) currently scores `5.55 / 10.50 = 52.9%` on form. Someone with a 100% win rate should not be sitting at a C grade.

## Goal

Make the form score represent "how well the player has performed in games they actually played" — independent of how many games their cohort has played.

## Fix

In `wprScore`, skip `'-'` slots when building `maxFormScore`:

```ts
// Before
const maxFormScore = formChars.reduce((acc, _, i) => acc + 3 * (1 - i * 0.15), 0)

// After
const maxFormScore = formChars.reduce(
  (acc, c, i) => c === '-' ? acc : acc + 3 * (1 - i * 0.15),
  0,
)
```

The existing `maxFormScore > 0 ? (rawFormScore / maxFormScore) * 100 : 0` guard still handles the all-dashes case (returns `0` rather than dividing by zero).

## Before / after examples

| `recentForm` | Current form score | New form score |
|---|---|---|
| `WW---` | 52.9 | 100.0 |
| `W----` | 28.6 | 100.0 |
| `WLD--` | 35.2 | 48.4 |
| `LL---` | 0.0 | 0.0 |
| `WWDLW` | 71.0 | 71.0 |
| `-----` | 0.0 | 0.0 |

Players with full 5-game histories (`WWDLW`, `LLLLL`, etc.) see no change — when every slot is played, the old and new denominators are identical.

## Test plan

Edit `lib/__tests__/utils.wpr.test.ts`:

**New test cases (assert against the new formula):**

- `recentForm = 'WW---'` with otherwise neutral inputs → form component contributes as if 100% form.
- `recentForm = 'W----'` → form = 100%.
- `recentForm = 'WLD--'` → form ≈ 48.4% (recompute exact under new formula).
- `recentForm = 'LL---'` → form = 0%.
- `recentForm = '-----'` → form = 0% (no NaN, no error thrown).

**Updates to existing tests:**

- Any hardcoded assertion using a `recentForm` that contains `'-'` characters — recompute the expected value under the new denominator.

**Regression coverage to confirm:**

- Full 5-game forms (`'WWDLW'`, `'LLLLL'`, `'WWWWW'`, `'DDDDD'`) produce unchanged values.
