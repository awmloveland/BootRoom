# 2.3 Refactor Pair-Pinning with a Placement Helper — Spec
_Date: 2026-04-21_
_Umbrella: [../2026-04-21-team-building-review-design.md](../2026-04-21-team-building-review-design.md)_
_Status: Ready to implement_

## Problem

`autoPick`'s pair-pinning loop (lib/autoPick.ts:65–108) has five parallel branches for "where is the associated player?":

1. Associated player is in the free search pool → normal case, toggle-driven.
2. Associated player is pinned as Team A's GK → put guest on Team A.
3. Associated player is pinned as Team B's GK → put guest on Team B.
4. Associated player is already in `pinnedTeamA` (placed by a prior pair) → put guest on Team A.
5. Associated player is already in `pinnedTeamB` (placed by a prior pair) → put guest on Team B.
6. Otherwise — associated player absent → skip (guest goes into the free pool).

Branches 2–5 are structurally identical: remove guest from free pool, push to the appropriate `pinnedTeam*` array. Only branch 1 flips `pairTeamToggle`. Reading the logic means walking through all five branches in order to notice the subtlety.

## Goal

Collapse the "where is the associated player?" lookup into a small pure helper. The main loop becomes: "find the associated player's team; place the guest there." One call, one branch.

## Fix

### Add a pure helper to `lib/autoPick.ts`

```ts
/** Return the team the associated player is already pinned to, or null if absent. */
function findAssocTeam(
  assocName: string,
  pinnedA: Player | null,
  pinnedB: Player | null,
  pinnedTeamA: Player[],
  pinnedTeamB: Player[],
): 'A' | 'B' | null {
  if (assocName === pinnedA?.name) return 'A'
  if (assocName === pinnedB?.name) return 'B'
  if (pinnedTeamA.some((p) => p.name === assocName)) return 'A'
  if (pinnedTeamB.some((p) => p.name === assocName)) return 'B'
  return null
}
```

### Rewrite the pair-pinning loop

```ts
for (const [guestName, assocName] of (pairs ?? [])) {
  const guest = searchPool.find((p) => p.name === guestName)
  if (!guest) continue

  const assoc = searchPool.find((p) => p.name === assocName)

  if (assoc) {
    // Normal case: both in free pool. Assign by toggle, then flip.
    searchPool = searchPool.filter((p) => p !== guest && p !== assoc)
    if (pairTeamToggle) {
      pinnedTeamA.push(guest, assoc)
    } else {
      pinnedTeamB.push(guest, assoc)
    }
    pairTeamToggle = !pairTeamToggle
    continue
  }

  // Associated player already pinned elsewhere — follow them.
  const team = findAssocTeam(assocName, pinnedA, pinnedB, pinnedTeamA, pinnedTeamB)
  if (team === 'A') {
    searchPool = searchPool.filter((p) => p !== guest)
    pinnedTeamA.push(guest)
  } else if (team === 'B') {
    searchPool = searchPool.filter((p) => p !== guest)
    pinnedTeamB.push(guest)
  }
  // Otherwise: associated player absent entirely. Skip; guest stays in searchPool.
}
```

### Critical behaviour to preserve

- **`pairTeamToggle` flips only in the "associated player found in free pool" case.** Follower cases (GK-pinned or already-pair-placed) do not advance the toggle.
- **Guests with absent associated players stay in `searchPool`.** Do not remove them — they get distributed freely by the subsequent split search.
- **Pool mutations and team placements are byte-for-byte identical** to the 5-branch version.

## Test plan

No new behavioural tests — pure refactor. All existing pair tests must pass unchanged:

- `autoPick — one guest+associated pair`
- `autoPick — two guests sharing one associated player`
- `autoPick — associated player is a GK`
- `autoPick — associated player not in squad`
- `autoPick — guest has goalkeeper: true`

**New unit tests for the helper** (in `lib/__tests__/autoPick.test.ts`; export `findAssocTeam` from `lib/autoPick.ts`):

- `findAssocTeam('Alice', null, null, [], [])` → `null`
- `findAssocTeam('Alice', { name: 'Alice', ... }, null, [], [])` → `'A'`
- `findAssocTeam('Alice', null, { name: 'Alice', ... }, [], [])` → `'B'`
- `findAssocTeam('Alice', null, null, [{ name: 'Alice', ... }], [])` → `'A'`
- `findAssocTeam('Alice', null, null, [], [{ name: 'Alice', ... }])` → `'B'`
- **Precedence:** if `pinnedA.name === 'Alice'` and `pinnedTeamA` also contains `'Alice'`, returns `'A'` (the pinned-GK check runs first).
