You are adding an optional seedable RNG parameter to the BootRoom 5-a-side league app's auto-pick algorithm so tests can run deterministically instead of retrying or tolerating randomness.

**Working directory:** `/Users/willloveland/conductor/workspaces/bootroom/almaty`
**Branch:** `awmloveland/team-ratings-exploration`

## What you're changing

`autoPick` currently uses `Math.random()` directly in 3–4 places (GK shuffle, random-sample fallback, tolerance-pool shuffle, and — after 1.4 — odd-player coin flip). Tests that exercise these paths retry up to 20 times or use 100-run statistical bounds to tolerate randomness.

Fix: add an optional `random?: () => number` parameter to `autoPick`. Production calls pass nothing and use `Math.random()` as today. Tests pass a seeded LCG for determinism.

**No behavioural change in production.**

## How to approach it

1. **Create the helper first.** Add `lib/__tests__/helpers/seeded-rng.ts` with the LCG from the spec. Add a test file (or colocated tests) asserting:
   - Fixed sequence for seed 42.
   - Two instances with same seed produce identical sequences.
   - Different seeds produce different sequences.
   - All outputs lie in `[0, 1)`.

2. **API change.** Open `lib/autoPick.ts`. Add `random?: () => number` as the fourth parameter. Inside the function body, add `const rng = random ?? Math.random`.

3. **Substitutions.** Replace every `Math.random()` call inside `autoPick` with `rng()`. Sites:
   - GK pool shuffle.
   - Random-sample fallback (inside the `for (let i = 0; i < FALLBACK_SAMPLE_COUNT; i++)` loop).
   - Tolerance pool shuffle before picking suggestions.
   - (If 1.4 has landed) The `extraSlotToA` coin flip.

4. **Migrate flaky tests** in `lib/__tests__/autoPick.test.ts`:
   - Find the `rating-aware` test's 20-iteration for-loop. Replace with a single call using `seededRng(SEED)` — pick a SEED empirically (try a few until you get a split that demonstrates the "one strong + one weak per team" property). Assert the specific composition.
   - Find the `swap deduplication` test. Replace the 20-run loop with a handful of deterministic seeded runs; each asserts no pair of suggestions is a team-swap of another.

5. Run the full suite: `npm test`. All existing tests should pass. Newly-deterministic tests should pass without retries.

6. Run typecheck: `npx tsc --noEmit`.

## Constraints

- **Do not** change the default behaviour when `random` is undefined. Production must use `Math.random()` identically to today.
- **Do not** substitute `Math.random()` calls outside `autoPick`. Scope is autoPick only.
- **Do not** make the seed itself a parameter of `autoPick`. The caller constructs their own RNG.
- **Do not** change any test that isn't currently flaky. Property-based assertions (e.g., "total player count equals n") stay as-is.
- **Do not** commit the LCG as production code. It lives under `lib/__tests__/helpers/`.
