# 1.3 Exclude GK Mentality from Variety Bonus — Spec
_Date: 2026-04-21_
_Umbrella: [../2026-04-21-team-building-review-design.md](../2026-04-21-team-building-review-design.md)_
_Status: Ready to implement_

## Problem

`ewptScore` in `lib/utils.ts` awards a `+2` variety bonus when a team has 3 or more distinct mentalities (around line 182). `'goalkeeper'` is one of the four valid mentality values, so any team with a keeper + two distinct outfielder styles already has 3 mentalities and earns the bonus — even if the outfield has no real tactical mix. The keeper is already rewarded separately by the GK modifier (+0.5 to +2.5), so counting it toward variety is a double-count.

The code comment states the bonus is for "tactical variety", which reasonable reading interprets as variety among outfielders — not "you have a keeper."

## Goal

The variety bonus should reflect tactical variety **among outfielders**. Keepers are already credited via `gkModifier` and should not contribute to the variety count.

## Fix

Filter keepers out of the set before counting distinct mentalities. Because the codebase currently has two representations of "is a keeper" (`mentality === 'goalkeeper'` and the `goalkeeper: boolean` flag — item 2.1 will consolidate these later), filter on both fields defensively:

```ts
// Before
const mentalities = new Set(players.map((p) => p.mentality))
const varietyBonus = mentalities.size >= 3 ? 2 : 0

// After
const outfielders = players.filter((p) => p.mentality !== 'goalkeeper' && !p.goalkeeper)
const mentalities = new Set(outfielders.map((p) => p.mentality))
const varietyBonus = mentalities.size >= 3 ? 2 : 0
```

## Before / after examples

| Composition | Old variety count | Old bonus | New outfielder mentality count | New bonus |
|---|---|---|---|---|
| GK + 4 balanced | 2 | 0 | 1 | 0 |
| GK + 3 balanced + 1 attacker | 3 | **+2** ❌ | 2 | 0 ✓ |
| GK + balanced + attacker + defender | 4 | +2 | 3 | +2 ✓ |
| GK + balanced + balanced + attacker + defender | 4 | +2 | 3 | +2 ✓ |
| balanced + attacker + defender + 2 others (no GK) | 3 | +2 | 3 | +2 |
| GK + attacker + defender (3 players) | 3 | +2 | 2 | 0 ✓ |

Rule summary: **the bonus now depends only on outfielder mentality diversity.** Teams with {keeper + 2 distinct outfielder styles} lose the spurious bonus; all other cases are unchanged.

## Test plan

**New tests in `lib/__tests__/utils.wpr.test.ts`:**

- Team of `{GK, balanced, balanced, balanced, attacker}` → variety bonus = 0.
- Team of `{GK, balanced, attacker, defender}` → variety bonus = 2.
- Team of `{GK, balanced, balanced, balanced, balanced}` → variety bonus = 0.
- Team of `{balanced, attacker, defender, balanced, attacker}` (no GK) → variety bonus = 2.
- Team of `{GK, attacker, defender}` (3 players, 2 outfielder mentalities) → variety bonus = 0.

**Edge case (dual-representation guard):**

- Team with a player where `mentality !== 'goalkeeper'` but `goalkeeper: true` (mismatched flags) → that player is still excluded from the variety count. Verifies the defensive filter during the item-2.1 transition.

**Updates to existing tests:**

- Walk every `ewptScore` test. For each team composition, check whether its old variety count included a GK. If yes, recompute the expected score without the `+2` (or with the correct bonus under the new rule).

## Out of scope

- Consolidation of the dual keeper representation (`goalkeeper: boolean` vs `mentality === 'goalkeeper'`). That's item 2.1. This spec filters on both defensively.
- Changes to the GK modifier, depth modifier, `avgWpr` weight, or top-2 weight.
- Changes to the mentality enum (`Mentality` type in `lib/types.ts`) — the four-way enum is preserved.

## Interaction with other items

- **1.2** changes the weights surrounding this term. Both can land independently; their effects compose cleanly.
- **2.1** will simplify the filter to `p.mentality !== 'goalkeeper'` once the `goalkeeper: boolean` field is removed.
